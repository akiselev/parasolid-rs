1.1 Introduction

This document describes functional enhancements and interface changes that have been added to Parasolid V35.0, and gives information on any other matters that are specific to this release.

[back to top]

1.2 Lattices

This section describes improvements that have been made to Parasolid’s support for representing lattices in Parasolid V35.0. Full details on Parasolid lattices can be found in Chapter 20, “Lattice Geometry”, of the Parasolid Functional Description.

[back to top]

1.2.1 Creating a patterned rod and ball lattice

In Parasolid V35.0 you can now create a complete lattice by applying a pattern to an initial portion of the lattice. You can optionally supply a user-defined callback function that can deform the pattern.

 

Figure 1-1 Creating a patterned lattice using a supplied lattice

For more information, see Section 20.6, “Creating a lattice structure from a core lattice” in the Parasolid Functional Description.

Interface changes

 

New

PK_LATTICE_make_patterned, PK_LATTICE_make_patterned_o_t, PK_LATTICE_make_patterned_r_f, PK_LATTICE_make_patterned_r_t

 

[back to top]

1.2.2 Creating a lattice-like body from a mesh body

In Parasolid V35.0, you can pattern a facet body into a lattice-like complex mesh body. This is achieved by transforming instances of the facet body and deforming the result using a user-defined callback function to match a rectilinear pattern.

 

Figure 1-2 Creating a lattice-like body from a mesh body

For more information, see Section 20.7, “Creating a lattice-like mesh body” in the Parasolid Functional Description.

Interface changes

 

New

PK_BODY_make_patterned, PK_BODY_make_patterned_o_t, PK_BODY_make_patterned_r_f, PK_BODY_make_patterned_r_t, PK_BODY_patterned_t, PK_pattern_bound_t, PK_pattern_cell_t, PK_SKEWBOX_sf_t, PK_pattern_t, PK_pattern_type_t, PK_pattern_rectilinear_o_t,

[back to top]

1.2.3 Creating blended lattices

In V35.0, you can now create blended lattices where the blend size of the ltopologies is the same throughout the lattice or where the blend size varies. Blending a lattice allows you to increase the stiffness of the lattice thereby improving its structural strength.

 

Figure 1-3 Blended lattice with ltopologies of different blend sizes

For more information, see Chapter 20.5, “Creating a blended lattice” in the Parasolid Functional Description.

 

Note: This functionality is not complete in the current release and the description above is included for information only. We expect full functionality to be available in a future release of Parasolid though you should note that the information here is subject to change.

Interface changes

 

Changed

PK_LATTICE_create_by_graph: new options; lball_blend_type and lball_blend_size

PK_LATTICE_graph_cyl_o_t, PK_LATTICE_graph_cone_o_t : new option lball_blend_sizes

 

[back to top]

1.2.4 Merging coincident ltopologies when clipping a lattice

You can choose whether coincident ltopologies that have been created as a result of clipping should be merged. This allows a cleaner clipping result.

 

Figure 1-4 Merging duplicate ltopologies when clipping a lattice

For more information, see Section 55.6.4.2, “Merging duplicate ltopologies when clipping lattices” in the Parasolid Functional Description.

Interface changes

 

New

PK_LTOPOL_dup_merge_t: used by merge_duplicates

Changed

PK_LATTICE_clip: new options; merge_duplicates and duplicate_tolerance

[back to top]

1.2.5 Finding the non-axis aligned box of a lattice

Parasolid V35.0 allows you to create a tight non-axis aligned box for a given lattice.

For more information, see Section 26.6.1.1, “Axis-aligned and non axis-aligned boxes” in the Parasolid Functional Description.

Interface changes

1.3.2 Support for models of different scales

Parasolid V35.0 provides support for models whose parts are of different scales by allowing geometry created at a larger scale to be scaled and trimmed so they can interact with entities at a smaller scale. This functionality can be useful when working with assemblies that consist of mixed scale parts.

For more information, see Section 14.9, “Representing models whose parts are of different scales” in the Parasolid Functional Description.

In addition, you can also generate facets for a set of models which are created at different scales

For more information, see Section 107.2.22, “Facet generation on bodies created at different scalesDOC2037” in the Parasolid Functional Description.

This section lists the new session switches that have been added to Parasolid V35.0.

Session switches should only be used when updating models created using an older version of Parasolid (including all versions prior to V28.1), and only when directed to do so by Parasolid Technical Support. These switches are not required when performing new modelling operations.

See Section 116.3, “Reproducing behaviour using the original system”, in the Parasolid Functional Description for more information.

1.16.1 Changes for Parasolid V35.0

This section describes changes to the platforms on which Parasolid ships that are relevant to the V35.0 release. For full details of changes to platforms, please see the customer letter accompanying the Parasolid V35.0 release.

  • We have changed the C++ code in Parasolid to being compiled as c++17 on all platforms.
  • The GCC 4.8 build of the Intel Linux platform has been dropped. The GCC 7.3 build is the only build on this platform.
  • We have replaced the Visual Studio 2019 build of ARM Win with a Visual Studio 2022 build
  • The ARM Android builds are now compatible with Android 9 and later and are built with NDK21b.

[back to top]

1.16.2 Plans for Parasolid V35.1

This section describes our current plans for Parasolid V35.1. For more details, please see the customer letter accompanying the Parasolid V35.1 release.

Parasolid V35.1 is expected to ship in the first quarter of 2023

We intend to replace:

  • The NDK21b build of ARM Android with one built on NDK23b, still compatible with Android 9 onwards.
  • The ARM MacOS and ARM iOS builds, built on macOS 11, with new builds on macOS 13, supporting macOS 11 and later and iPadOS 15 or later.
  • The Intel MacOS and Intel iOS builds, built on macOS 11 and supporting iPadOS 13 and later, with new builds on macOS 13 and supporting macOS 11 or later and iPadOS 15 and later. These builds are intended to set the final build standard for these platforms; we do not intend to update them again before they are dropped. Apple plans to stop selling Intel-based computers, so at some point, it will be necessary for us to give up building software for them. The date for this is not yet set.
